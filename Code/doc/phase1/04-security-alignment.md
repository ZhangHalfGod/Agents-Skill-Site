# 04 — 与现网安全策略对齐（摘录）

> 完整原文：`MechAssist-Fusion/extern-api/doc/阿里云_ECS_安全修复清单.md`  
> 本文只保留**对本站（agents-skill-site）有约束力**的条款，避免重复维护长文。

## 1. 必须遵守（本站上线门禁）

| # | 条款 | 本站落地 |
|---|------|----------|
| 1 | 安全组公网仅 80/443 | 本站**不申请**新的公网端口 |
| 2 | Node 只听本机 | serve/预览进程绑定 `127.0.0.1` |
| 3 | 对外只经 Nginx | 增加 `location` 或独立 server，不直出 Node 端口 |
| 4 | 可灰度、可验证、可回滚 | 保留上一版 `dist`/git tag；`pm2 restart` 可回退 |
| 5 | 部署 URL 不进 standards 模板 | 只写 `Code/doc` + Nginx 配置 |

## 2. 现网已完成（可依赖）

- 安全组收敛 80/443；SSH 非全网开放（控制台登录）  
- `mechassist-api` → `127.0.0.1:3002`  
- Anti-DDoS 基础防护智能模式  

## 3. 现网仍待办（本站不阻塞，但上线时注意）

- SSH 仅密钥、关密码登录  
- Nginx 日志保留 ≥30 天与审查流程  

## 4. 每次发布本站时勾选

- [ ] `ss` 确认本站端口仅为 `127.0.0.1`  
- [ ] 安全组未误开本站端口  
- [ ] `nginx -t` 通过后 reload  
- [ ] 公网只验证 80/443 路径，不验证直连 3010  
- [ ] 回滚：上一 `dist` 或 git 版本可 5 分钟内恢复
