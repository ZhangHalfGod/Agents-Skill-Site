# Waterfall / Agile Mode Details

> Long-form reference; default execution uses the main skill page checklist only.

## Waterfall mode

- Execute strictly: **requirements → design → generation → test → release**
- Do **not** enter the next phase until the previous phase is approved
- Deliverables must include: **phase identifier**, **review status**, **owner**
- Tasks touching core layer / gateway / security / architecture must append the **Core Module AI Generation Red Record**

## Agile mode

- Execute in Sprint cycles
- Daily standup syncs AI generation progress
- Sprint retrospective optimizes Prompts
- Deliverables must include: **phase identifier**, **review status**, **owner**
- Sprint items touching core layer / gateway / security / architecture must complete the **Core Module AI Generation Red Record** within the current Sprint
